**Вопрос 1. Перечислите основные функциональные компоненты стенда SDK-1.1M и кратко опишите функции, выполняемые каждым
элементом.**

- Процессорный модуль (например, STM32F407VIT6, которого использовали в работах) — это центральный контроллер, который
  выполняет пользовательский код, управляет периферией, а также отвечает за логику, обмен через шины, обработку событий
  и прерывания.
- Внешняя память (EEPROM, 1Кб) — позволяет хранить данные вне RAM/Flash MCU — например, параметры, настройки, постоянные
  данные, которые остаются после выключения питания.
- Набор светодиодов — для визуальной индикации (статусы, сигналы, отладка)
- RTC-модуль (часы реального времени, к примеру MCP79411) — дает возможность отслеживать время/дату, использовать в
  приложениях с реальным временем (логирование, расписания, таймеры, события по времени)
- Клавиатура — для пользовательского ввода
- GPIO-расширитель (PCA9538PW) — обеспечивает множество универсальных входов/выходов: можно подключать кнопки,
  переключатели, датчики и дополнительные внешние устройства.
- Коммуникационные интерфейсы (RS-485, Ethernet 10/100, USB) — для связи стенда с внешним оборудованием, компьютером,
  сетью
- Графический OLED-дисплей — обеспечивает вывод информации в виде текста, графика, который является удобным с точки
  зрения UX — визуальный интерфейс, меню, отображения данных, статуса, UI.

**Вопрос 2. Перечислите основные программные и аппаратные компоненты, которые участвуют в передаче данных по интерфейсу
UART в режиме прерываний с прикладного уровня программы стенда SDK-1.1M в терминальную программу (например, PuTTY) на
компьютере и обратно. Кратко опишите роль этих компонентов в передаче данных.**
Программные компоненты:

- HAL / драйвер UART — отвечает за настройку UART: скорость бода, формат, включение прерывании, управление передачей и
  приемом. Также обеспечивает функции для чтения/записи байт, запуска приема в interrupt-режиме, управления состоянием
  передачи
- Обработчик прерывания — при приходе события UART прерывание вызывает ISR, который считывает байт из регистра, помещает
  его в программный буфер (например, кольцевой), или, при передаче, забирает следующий байт из буфера и записывает в
  регистр для отправки
- Программная очередь / кольцевой буфер — позволяет "буферизовать" данные: ISR — отправляющий пишет байты, основная
  программа — принимающий читает их по мере готовности; так приёмы/передачи не теряются, даже если главный цикл занят
- Функции начала/повторного старта приема (например, HAL_UART_Receive_IT()) — после обработки предыдущего байта ISR
  обычно вызывает (или драйвер сам автоматически перезапускает) приём для следующего байта — чтобы UART продолжал
  слушать вход

Есть ещё и сама программа (основной цикл), а также терминал.

Аппаратные компоненты:

- U(S)ART-модуль в MCU — "ядро" интерфейса: он превращает байты в серию битов с битами начала/остановки, управляет
  тактованием согласно настроенному скорости передачи данных (бод), а также обеспечивает прием (RX) и передачу (TX)
  данных
- Шины MCU для UART (TX, RX, GND) — физическое соединение между MCU и внешним устройством (USB-UART конвертером или
  COM-портом): линия передачи и приема (TX, RX), а также общий земляной сигнал (GND)
- USB-UART адаптер - преобразует сигналы UART в формат, понимаемый компьютером — USB и драйвера, и наоборот; делает
  возможным соединение между MCU и компьютера
- контроллер прерываний MCU (NVIC, внутренний контроллер прерываний) — обеспечивает, что при событии UART (прием байта,
  пустой буфер передачи и др.) генерируется аппаратное прерывание, которое передает управление обработчику (ISR).

**Вопрос 3. Перечислите основные программные и аппаратные компоненты, которые участвуют в получении кодов кнопок,
нажатых на матричной клавиатуре стенда SDK-1.1M, прикладным уровнем программы стенда. Кратко опишите роль этих
компонентов в процессе получения кодов нажатых кнопок.**

Программные компоненты (сначала инициализируются GPIO-пины):

- Функция сканирования — производит "опрос" по каждым строкам (опрашиваемая строка имеет значение логического уровня 0,
  остальные - 1), внутри которого производит опрос по столбцам (если кнопка нажата — то имеет значение логического
  уровня 0). По пересечению нулей можно определить, какая кнопка нажата
- Антидребезг — после физического нажатия контакты могут "дребезжать" — мигать замыкания-размыкания. Обычно реализуют
  программную задержку или фильтр: считывают несколько раз, и если кнопка стабильно нажата — фиксируют нажатие
- Буферизация событий — вместо немедленной реакции, код клавиатуры часто кладёт "событие нажатия" в буфер (кольцевой),
  чтобы основная логика могла потом его обработать, не блокируясь на сканировании
- Обработчик событий клавиатуры — при обнаружении нового нажатия переводит событие в символ и передаёт это событие в
  прикладной код.
  Есть ещё и сама программа (основной цикл).

Аппаратные компоненты:

- Матричная клавиатура — набор кнопок, соединённых строками и столбцами; при нажатии одной кнопки замыкается
  соответствующая пара (ряд-столбец), что создаёт электрическое соединение
- GPIO-пины MCU, к которым подключены ряды клавиатуры — пины настраиваются как выходы — MCU поочерёдно "опрашивает" по
  одному ряду, чтобы проверить, не нажата ли кнопка в этом ряду
- GPIO-пины MCU, к которым подключены столбцы — пины настраиваются как входы (обычно с подтяжкой вверх), их читают,
  чтобы определить, какой столбец "провалился" (сигнал 0) — это даёт номер столбца нажатой кнопки
- Питание и общая "земля" — для работы клавиатуры необходима общая схема: чтобы при замыкании кнопки течение тока шло от
  строки до столбца, потом "земли" (или наоборот, в зависимости от логики)
